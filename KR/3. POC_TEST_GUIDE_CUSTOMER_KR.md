## 주요 테스트 시나리오

> **프론트(사용자)와 관리자(Admin) 테스트를 동시에 진행하려면?**
>
> - 같은 브라우저 세션에서는 한 계정만 로그인할 수 있습니다.
> - 프론트와 어드민을 동시에 테스트하려면, "다른 브라우저" 또는 "시크릿/인프라이빗 창"을 각각 사용해 별도로 로그인하세요.

### [공통] 회원가입 및 로그인

- 사이트 우측 상단의 로그인/회원가입 버튼을 클릭해 직접 계정을 생성하거나, 위의 테스트 계정으로 로그인할 수 있습니다.

### 1. 일반 사용자(고객) 체험

- 제품 목록/상세 페이지에서 상품을 둘러보고 장바구니에 담아 결제까지 진행해보세요.
- Stripe 신용카드 결제(테스트 카드 사용 가능) 흐름을 확인할 수 있습니다.
- (선택) 브라우저/기기 환경이 지원하는 경우, 체크아웃에서 **Google Pay / Apple Pay(지갑 결제)** 도 테스트해볼 수 있습니다. 지갑 결제 노출 여부는 브라우저/기기, HTTPS, Stripe 도메인 등록 등에 따라 달라질 수 있습니다.
- 주문 내역, 견적 요청, 자동 이메일 안내 등도 체험 가능합니다.
- **Customer Support(문의/티켓) 체험**
  - 주문 완료 후 **주문 상세 페이지**에서 `Contact Support`로 문의(티켓)를 생성해보세요.
  - 우측 상단 사용자 메뉴의 `Customer Support`에서 티켓 목록/상세 및 CS 답변 스레드를 확인할 수 있습니다.

#### 상품 탐색 UX 테스트(권장)

- 상단 **Products**에서 아래를 확인해보세요.
  - 카테고리 트리 탐색(데스크톱 기준 좌측 사이드바)
  - 검색(상단 고정 바) 및 URL 동기화 필터(북마크/공유 가능)
  - 헤더 검색(돋보기 아이콘): 전체 화면 검색 모달에서 실시간 추천/결과 확인

### 2. 유통사(Distributor) 체험

- 유통사 계정으로 로그인하면 맞춤 가격, 할인, 전용 대시보드 등 유통사 전용 기능을 확인할 수 있습니다.
- 대량 구매 할인, 30일 외상 결제(PO), 견적 요청 등 B2B 기능을 직접 테스트해보세요.
- 견적 요청은 각 제품 상세 화면(페이지) 중간에 위치한 입력 UI에서 바로 진행할 수 있습니다.
- 유통사별 브랜드(로고/색상) 적용 기능도 구현되어 있으나, 실제 적용은 서브도메인 기반으로 동작합니다.
- PoC 데모 사이트는 단일 도메인 환경이므로 브랜드별 디자인 자동 적용은 직접 체험이 어렵습니다.

### 3. 관리자(Admin) 체험

- 관리자 계정으로 로그인하면 전체 주문/회원/제품/유통사 관리, 가격 정책 설정, 도메인/브랜딩 관리 등 관리자 전용 기능을 사용할 수 있습니다.
- White-label(브랜드별 쇼핑몰) 관리, 유통사별 가격/할인/도메인 설정, 자동 이메일 템플릿 관리 등 고급 기능을 확인해보세요.
- **제품 운영(권장 테스트 포인트)**
  - Admin 사이드바의 **Products** 메뉴에서 제품 상태(`draft/active/archived`)와 **현재 가격/할인가 + Next price(다음 적용 가격/시각)**를 확인할 수 있습니다.
  - **Products → 특정 제품 상세 화면**에서:
    - `Scheduled Sales`에서 세일을 **추가/수정/삭제**해보고, 상단 `Current/Next (effective)` 요약이 어떻게 바뀌는지 확인해보세요.
    - `Inventory History`에서 재고를 **증감(+/-) + 사유**로 조정하고 이력이 쌓이는지 확인해보세요.
- **Customer Support(티켓) 관리**
  - Admin 사이드바의 **Support Tickets** 메뉴에서 티켓 큐를 확인하고, 상태 변경/내부 노트 기록/답변(스레드) 작성이 가능합니다.
  - 티켓 상세에서 주문이 연결된 경우 `Show Order` 토글로 주문 요약을 같은 화면에서 확인할 수 있습니다.

- Admin settings: 관리자 페이지의 **Settings**에서 PO/Quote 등 테스트 이메일 수신자(`ADMIN_EMAIL`)를 설정할 수 있습니다.
- 운영 설정(Settings) 테스트:
  - Admin 사이드바의 **Settings**에서 **최소 주문금액/배송비/무료배송 기준/저재고 임계치**를 변경해보고
  - 프론트의 Cart/Checkout에서 값이 반영되는지 확인해보세요(배송비/최소 주문금액 체크).
- 재고 운영 테스트:
  - Admin 사이드바의 **Inventory** 메뉴에서 저재고 리포트를 확인하고 CSV로 export 해보세요.
  - 템플릿 CSV를 내려받아 `sku,stock,reason,note`로 재고를 일괄 업데이트해보고, 변경 결과/로그가 남는지 확인해보세요.
- 주문 운영 테스트:
  - Admin 사이드바의 **Orders** 메뉴에서 주문 CSV export를 실행해보고(기간 필터 포함), CSV가 아이템 단위로 출력되는지 확인해보세요.
  - **Orders → 주문 상세 화면**에서 택배사/송장번호를 입력하고 상태를 shipped/delivered로 변경했을 때 발송/배송완료 시간이 기록되는지 확인해보세요.
  - (대량) **Orders → Operations → Bulk fulfillment update (CSV)**로 택배사/송장번호를 일괄 업로드해보세요.
  - 고객 화면 주문 상세에서 **배송조회(Track shipment)** 링크와 **Shipping fee** 항목이 노출되는지 확인해보세요.

- 참고: 일부 고급 기능(예: 이메일 템플릿 직접 편집, 모든 White-label 디자인 자동화, 유통사별 완전 독립 관리자 모듈 등)은 PoC 단계에서 제한적이거나 일부만 구현되어 있을 수 있습니다. 대부분의 관리자 기능은 admin@test.com 계정으로 실제로 체험할 수 있습니다.

## 추가 안내

- PoC 사이트는 테스트용으로, 실제 결제/배송이 이루어지지 않습니다.
- 테스트 중 궁금한 점이나 오류가 있으면 언제든 문의해 주세요.
- 다양한 계정/역할로 자유롭게 체험해보고, 새로운 계정도 직접 만들어보시길 권장합니다.

### 이메일 수신 확인 안내

- 주문/견적/배송 등 주요 이벤트 발생 시, 회원가입 시 입력한 이메일 주소로 안내 메일이 발송됩니다.
- 관리자(Admin) 테스트 시, 관리자 페이지의 **Settings**에서 `ADMIN_EMAIL`을 본인 이메일로 설정하면 테스트 알림 메일을 직접 받아볼 수 있습니다.
- Gmail 등 주요 메일 서비스에서 스팸함도 함께 확인해 주세요.

### Stripe 테스트 카드 번호 안내

- 결제 테스트 시, 아래 Stripe 테스트 카드 번호를 사용할 수 있습니다.
  - 카드번호: `4242 4242 4242 4242`
  - 만료일: 아무 미래 날짜 (예: 12/34)
  - CVC: 아무 3자리 숫자 (예: 123)
  - 우편번호: 아무 5자리 숫자 (예: 12345)
- 실제 결제는 발생하지 않으니 안심하고 테스트하세요.
